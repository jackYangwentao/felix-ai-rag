# 文档增量导入设计方案

## 1. 背景与问题

### 当前问题
- 系统使用 `InMemoryEmbeddingStore`，每次导入文档时直接追加，不做去重
- 没有版本管理机制，无法识别文档是否已更新
- 需要重启应用才能清空向量库，或手动管理
- 文档更新后需要重新全量导入，效率低下

### 目标
- 实现文档增量导入，避免重复处理未变化的文档
- 支持文档版本管理，识别内容变化
- 提供文档更新、删除能力
- 保持向后兼容性

## 2. 设计方案

### 2.1 核心思路

采用 **文档指纹 + 版本管理** 的方案：

1. 为每个文档生成唯一标识（documentId）
2. 计算文档内容哈希（contentHash）作为版本指纹
3. 导入时检查是否已存在相同文档
4. 根据哈希判断是否内容变化，决定更新策略

### 2.2 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                     文档增量导入流程                          │
└─────────────────────────────────────────────────────────────┘

用户上传文档
    ↓
计算文档指纹（MD5/SHA256）
    ↓
检查文档是否存在
    ├─ 不存在 → 新文档导入流程
    │              ↓
    │         分块 → 嵌入 → 索引
    │              ↓
    │         记录文档元数据（ID + 哈希 + 时间）
    │
    └─ 已存在 → 比较内容哈希
                   ↓
            ├─ 哈希相同 → 跳过（文档未变化）
            │
            └─ 哈希不同 → 更新流程
                           ↓
                      删除旧分片
                           ↓
                      插入新分片
                           ↓
                      更新元数据
```

## 4. 接口设计

### 4.1 增量导入接口

```java
/**
 * 增量文档导入
 * 自动检测文档变化，只处理有变化的文档
 */
@PostMapping("/documents/incremental")
public ResponseEntity<IncrementalImportResult> uploadDocumentIncremental(
        @RequestParam("file") MultipartFile file,
        @RequestParam(value = "documentId", required = false) String documentId,
        @RequestParam(value = "forceUpdate", defaultValue = "false") boolean forceUpdate) {
    
    // 1. 生成或获取文档ID
    if (documentId == null) {
        documentId = generateDocumentId(file);
    }
    
    // 2. 计算内容哈希
    String contentHash = calculateContentHash(file);
    
    // 3. 检查是否已存在
    DocumentMetadata existing = documentRegistry.get(documentId);
    
    if (existing != null) {
        // 3.1 检查内容是否变化
        if (existing.getContentHash().equals(contentHash) && !forceUpdate) {
            return ResponseEntity.ok(IncrementalImportResult.builder()
                    .documentId(documentId)
                    .status("SKIPPED")
                    .message("文档未变化，跳过导入")
                    .build());
        }
        
        // 3.2 删除旧版本
        deleteDocumentChunks(existing);
    }
    
    // 4. 导入新版本
    DocumentMetadata newMetadata = indexDocumentWithMetadata(file, documentId, contentHash);
    
    return ResponseEntity.ok(IncrementalImportResult.builder()
            .documentId(documentId)
            .status(existing != null ? "UPDATED" : "CREATED")
            .version(newMetadata.getVersion())
            .chunkCount(newMetadata.getChunkCount())
            .message("文档导入成功")
            .build());
}
```

### 4.2 批量增量导入

```java
/**
 * 批量增量导入
 * 支持文件夹批量导入，自动识别新增、更新、删除的文档
 */
@PostMapping("/documents/incremental/batch")
public ResponseEntity<BatchIncrementalResult> batchUploadIncremental(
        @RequestParam("files") List<MultipartFile> files,
        @RequestParam(value = "folderId") String folderId) {
    
    BatchIncrementalResult result = BatchIncrementalResult.builder()
            .totalFiles(files.size())
            .created(0)
            .updated(0)
            .skipped(0)
            .failed(0)
            .build();
    
    // 获取该文件夹下已有的文档列表
    Set<String> existingDocs = getDocumentsByFolder(folderId);
    Set<String> processedDocs = new HashSet<>();
    
    for (MultipartFile file : files) {
        try {
            String documentId = folderId + "/" + file.getOriginalFilename();
            IncrementalImportResult importResult = processIncrementalUpload(file, documentId);
            
            switch (importResult.getStatus()) {
                case "CREATED": result.incrementCreated(); break;
                case "UPDATED": result.incrementUpdated(); break;
                case "SKIPPED": result.incrementSkipped(); break;
            }
            
            processedDocs.add(documentId);
        } catch (Exception e) {
            result.incrementFailed();
            result.addError(file.getOriginalFilename(), e.getMessage());
        }
    }
    
    // 处理已删除的文档（在原有列表中但不在新列表中）
    existingDocs.removeAll(processedDocs);
    for (String deletedDocId : existingDocs) {
        deleteDocument(deletedDocId);
        result.incrementDeleted();
    }
    
    return ResponseEntity.ok(result);
}
```

```java
/**
 * 删除指定文档
 */
@DeleteMapping("/documents/{documentId}")
public ResponseEntity<String> deleteDocument(@PathVariable String documentId) {
    DocumentMetadata metadata = documentRegistry.get(documentId);
    if (metadata == null) {
        return ResponseEntity.notFound().build();
    }
    
    // 删除向量库中的分片
    deleteDocumentChunks(metadata);
    
    // 更新注册表
    documentRegistry.remove(documentId);
    
    return ResponseEntity.ok("文档删除成功: " + documentId);
}

/**
 * 检查文档状态
 */
@GetMapping("/documents/{documentId}/status")
public ResponseEntity<DocumentStatusResponse> checkDocumentStatus(
        @PathVariable String documentId,
        @RequestParam(value = "hash", required = false) String contentHash) {
    
    DocumentMetadata metadata = documentRegistry.get(documentId);
    
    if (metadata == null) {
        return ResponseEntity.ok(DocumentStatusResponse.builder()
                .exists(false)
                .status("NOT_FOUND")
                .build());
    }
    
    boolean changed = contentHash != null && !contentHash.equals(metadata.getContentHash());
    
    return ResponseEntity.ok(DocumentStatusResponse.builder()
            .exists(true)
            .status(metadata.getStatus().name())
            .version(metadata.getVersion())
            .indexTime(metadata.getIndexTime())
            .changed(changed)
            .build());
}
```

## 5. 核心实现

### 5.1 文档指纹计算

```java
@Service
public class DocumentFingerprintService {
    
    /**
     * 计算文件内容哈希（MD5）
     */
    public String calculateHash(MultipartFile file) throws IOException {
        return DigestUtils.md5Hex(file.getInputStream());
    }
    
    /**
     * 计算文件内容哈希（SHA256，更安全）
     */
    public String calculateHashSha256(MultipartFile file) throws IOException {
        return DigestUtils.sha256Hex(file.getInputStream());
    }
    
    /**
     * 生成文档ID
     * 策略1：基于文件路径
     * 策略2：基于内容哈希（去重）
     * 策略3：UUID（随机）
     */
    public String generateDocumentId(MultipartFile file, String strategy) {
        switch (strategy) {
            case "PATH":
                return file.getOriginalFilename();
            case "CONTENT":
                try {
                    return calculateHash(file);
                } catch (IOException e) {
                    return UUID.randomUUID().toString();
                }
            case "UUID":
            default:
                return UUID.randomUUID().toString();
        }
    }
}
```

### 5.2 向量库分片管理

```java
@Service
public class VectorStoreManager {
    
    @Autowired
    private EmbeddingStore<TextSegment> embeddingStore;
    
    /**
     * 删除指定文档的所有分片
     * 
     * 注意：InMemoryEmbeddingStore 不支持直接删除，
     * 需要重新构建或切换到支持删除的数据库（Redis/Qdrant）
     */
    public void deleteDocumentChunks(DocumentMetadata metadata) {
        List<String> chunkIds = metadata.getChunkIds();
        
        if (chunkIds != null && !chunkIds.isEmpty()) {
            for (String chunkId : chunkIds) {
                // 根据存储类型执行删除
                if (embeddingStore instanceof DeletableEmbeddingStore) {
                    ((DeletableEmbeddingStore) embeddingStore).delete(chunkId);
                }
            }
        }
    }
    
    /**
     * 索引文档并记录分片ID
     */
    public DocumentMetadata indexDocument(String content, DocumentMetadata metadata) {
        // 分块
        TextChunker chunker = chunkerFactory.createChunkerForFile(metadata.getDocumentName());
        List<String> chunks = chunker.chunk(content);
        
        List<String> chunkIds = new ArrayList<>();
        
        // 索引每个分块
        for (int i = 0; i < chunks.size(); i++) {
            String chunk = chunks.get(i);
            TextSegment segment = TextSegment.from(chunk);
            
            // 添加元数据到 segment
            segment.metadata().put("document_id", metadata.getDocumentId());
            segment.metadata().put("chunk_index", String.valueOf(i));
            
            Embedding embedding = embeddingModel.embed(segment).content();
            String chunkId = embeddingStore.add(embedding, segment);
            chunkIds.add(chunkId);
        }
        
        // 更新元数据
        metadata.setChunkIds(chunkIds);
        metadata.setChunkCount(chunks.size());
        metadata.setVersion(metadata.getVersion() != null ? metadata.getVersion() + 1 : 1);
        metadata.setUpdateTime(LocalDateTime.now());
        
        return metadata;
    }
}
```

### 5.3 文档注册表持久化

```java
@Component
public class DocumentRegistry {
    
    private final Map<String, DocumentMetadata> registry = new ConcurrentHashMap<>();
    private final Path registryFile;
    
    public DocumentRegistry(@Value("${rag.registry.path:data/document-registry.json}") String path) {
        this.registryFile = Paths.get(path);
        loadRegistry();
    }
    
    /**
     * 从文件加载注册表
     */
    @PostConstruct
    public void loadRegistry() {
        if (Files.exists(registryFile)) {
            try {
                String json = Files.readString(registryFile);
                List<DocumentMetadata> documents = objectMapper.readValue(json, 
                        new TypeReference<List<DocumentMetadata>>() {});
                documents.forEach(doc -> registry.put(doc.getDocumentId(), doc));
                log.info("加载文档注册表: {} 个文档", registry.size());
            } catch (IOException e) {
                log.error("加载文档注册表失败", e);
            }
        }
    }
    
    /**
     * 保存注册表到文件
     */
    @PreDestroy
    public void saveRegistry() {
        try {
            Files.createDirectories(registryFile.getParent());
            String json = objectMapper.writeValueAsString(new ArrayList<>(registry.values()));
            Files.writeString(registryFile, json);
            log.info("保存文档注册表: {} 个文档", registry.size());
        } catch (IOException e) {
            log.error("保存文档注册表失败", e);
        }
    }
    
    public void register(DocumentMetadata metadata) {
        registry.put(metadata.getDocumentId(), metadata);
        saveRegistry();
    }
    
    public void remove(String documentId) {
        registry.remove(documentId);
        saveRegistry();
    }
    
    public DocumentMetadata get(String documentId) {
        return registry.get(documentId);
    }
    
    public List<DocumentMetadata> listDocuments(String folder) {
        return registry.values().stream()
                .filter(doc -> folder == null || doc.getDocumentId().startsWith(folder))
                .collect(Collectors.toList());
    }
}
```

## 6. 向量数据库选型建议

### 6.1 当前方案（InMemoryEmbeddingStore）

**优点**：
- 简单易用，无需外部依赖
- 启动快速，适合开发测试

**缺点**：
- 不支持删除操作
- 数据不持久化，重启丢失
- 无法处理大规模数据

**增量导入实现**：
- 标记删除而非物理删除
- 定期重建索引（全量重新导入）

### 6.2 推荐方案（Redis）

**优点**：
- 支持删除和更新操作
- 内存级性能，速度快
- 数据可持久化

**配置**：
```yaml
rag:
  vector-store:
    type: redis
    redis:
      host: localhost
      port: 6379
      index-name: rag-index
```

### 6.3 企业级方案（Qdrant）

**优点**：
- 高性能向量检索
- 支持元数据过滤
- 支持删除和更新
- 分布式部署

**配置**：
```yaml
rag:
  vector-store:
    type: qdrant
    qdrant:
      host: localhost
      port: 6334
      collection-name: rag-collection
```

## 9. 注意事项

1. **文档ID策略**：建议使用文件路径作为ID，便于管理和识别
2. **哈希算法**：MD5 足够用于变化检测，如需更安全可使用 SHA256
3. **并发处理**：注册表操作需要线程安全，使用 ConcurrentHashMap
4. **持久化**：定期保存注册表，防止数据丢失
5. **向量库选择**：生产环境建议使用 Redis 或 Qdrant，支持真正的删除操作

## 10. 总结

本方案通过 **文档指纹 + 版本管理** 实现了文档增量导入，主要优势：

- ✅ 避免重复处理未变化的文档，提高效率
- ✅ 支持文档更新和删除，便于管理
- ✅ 向后兼容，不影响现有功能
- ✅ 可扩展性强，支持多种存储后端
- ✅ 实现简单，可在现有架构上快速落地
